Section 1

What is Hermes Agent?

A plain-English overview of the project, its purpose, and why it matters — no technical background required.

The Big Picture

Hermes Agent is an open-source AI assistant framework built by Nous Research. Think of it as the engine behind a personal AI assistant that you own, control, and deploy anywhere — on your laptop, in a Docker container, on a remote server, or even as a serverless cloud function.

Unlike ChatGPT or Claude, which are services you access through a web browser, Hermes Agent is software you run yourself. You choose which AI model powers it (OpenAI, Anthropic Claude, Google Gemini, or any open-source model), where it runs, and what it can do.

💡
The Key Analogy

If ChatGPT is like renting an apartment (you use it but don't own it), Hermes Agent is like owning a house. You have full control: choose the layout, add rooms (tools/skills), and it gets smarter the longer you live there.

What Makes It Special?

🧠

It Learns From You

Unlike most AI tools that forget everything between conversations, Hermes builds a persistent memory of who you are, your preferences, and your past work sessions.

🛠️

It Can Take Action

Not just a chatbot — Hermes can browse the web, write and run code, manage files, send messages, schedule tasks, and control your computer through 40+ built-in tools.

📚

It Creates Its Own Skills

When Hermes solves a complex problem, it can save the solution as a reusable "skill" — so next time, it can do the same task faster and better without starting from scratch.

🌐

It Lives Everywhere

Chat with it via terminal, a web browser, Telegram, Discord, Slack, WhatsApp, or Signal. One agent, any interface — it works the same wherever you reach it.

The "Closed Learning Loop" Explained Simply

Most AI assistants work like this: you ask a question → they answer → they forget. Every conversation starts fresh. Hermes is different because it has a closed learning loop:

1
You give Hermes a task

e.g., "Research the top 5 competitors in our market and write a report."

2
Hermes completes the task using its tools

It searches the web, reads articles, synthesises the information, and writes the report.

3
It saves what it learned as a "skill"

The exact steps it followed are saved so next time you ask for competitor research, it's instant.

4
It updates your user profile

It remembers you prefer bullet-point summaries with links, not long paragraphs — and applies this preference automatically.

5
Next session, it's smarter

Memory + skills from past sessions are loaded in, so it picks up exactly where you left off.

Core Concepts (Non-Technical)

The Five Pillars of Hermes

  • 🤖
    The Agent — The Brain The core AI that understands your requests, plans what to do, uses tools to get things done, and crafts your final answer. Think of it as a very capable AI employee who can delegate tasks to specialists.
  • 🔧
    Tools — The Hands Specific capabilities the agent can use: searching the web, running code, reading/writing files, sending messages, browsing websites, generating images, and more. The agent picks the right tools for each task automatically.
  • 📖
    Skills — The Playbook Saved, reusable workflows for tasks you do repeatedly. Skills are like recipes — once written, you can run them with a single command (e.g., /competitor-research). Hermes ships with 27 categories of built-in skills.
  • 🧩
    Memory — The Long-Term Knowledge Store Hermes can remember facts, preferences, past conversations, and project context across sessions. Unlike standard chatbots, it won't ask you the same questions twice.
  • 🌉
    Gateway — The Delivery System The layer that lets you interact with Hermes from any messaging app (Telegram, Slack, Discord, WhatsApp, etc.) or interface (web, terminal, IDE). One AI brain, many access points.

How It Works — The Agent Loop

The technical heartbeat of Hermes: the conversation and tool-calling loop that drives everything.

The Core Loop (Technical)

Every interaction with Hermes runs through the AIAgent class defined in run_agent.py. This is the engine — roughly 12,000 lines of Python that orchestrate context, tool calls, provider communication, and response streaming.

Step-by-Step Breakdown

1
Context Assembly

Before sending anything to the AI model, Hermes builds a "context package" consisting of: a dynamic system prompt, relevant memories from past sessions, any active skills, the current conversation history, and optional context files (documents you've loaded). Token budgeting ensures this never exceeds the model's context window.

2
LLM API Call

The assembled context is sent to whatever AI model is configured. The model receives a list of "tool schemas" — descriptions of every available tool including what it does and what parameters it accepts. The model can then decide to either reply directly or call one or more tools.

3
Tool Execution (if needed)

If the model decides to use a tool, Hermes parses the tool call, validates the parameters, executes the tool (e.g., runs a web search, executes code, reads a file), and appends the result back to the conversation. The model then sees the result and decides: reply now, or call another tool?

4
Loop Until Done

The tool-calling loop repeats until the model produces a text-only response (no more tool calls). There's a configurable maximum iteration limit (default: 90 turns) to prevent runaway loops.

5
Response + Memory Update

The final text response is streamed back to the user. Optionally, the session is saved to SQLite with full-text search indexing so past conversations can be recalled in future sessions.

The Context Budget System

Every AI model has a "context window" — a maximum amount of text it can process at once. Hermes manages this carefully through the context compressor (agent/context_compressor.py):

⚖️
How Token Budgeting Works

When the conversation approaches the model's context limit, Hermes automatically summarises older messages, keeping the most recent context intact. This allows conversations to run indefinitely without losing coherence. The user never has to manually manage this — it happens silently in the background.

Extended Thinking (Reasoning Mode)

For Anthropic Claude models, Hermes supports extended thinking — a mode where the model explicitly reasons through a problem before answering. This is stored in the message history as "reasoning blocks" and gives you visibility into how the AI arrived at its answer.

# Example: Extended thinking stored in message history
{
  "role": "assistant",
  "reasoning": "Let me think through this step by step...",
  "content": "Based on my analysis, the answer is..."
}

Parallel Tool Execution

Hermes can run multiple tools simultaneously using the delegate_task tool, which spawns sub-agents. For example, when researching a topic, it might concurrently search five different web sources, read three files, and check a database — all at the same time, dramatically reducing wait times.

The Six Major Components

1. Core Agent Engine — run_agent.py

The ~12,000-line AIAgent class is the heart of the system. It manages the entire conversation lifecycle: building context, calling the LLM, dispatching tool calls, handling retries, and streaming responses. Key sub-systems it orchestrates:

  • Prompt Builder (agent/prompt_builder.py) — Assembles the system prompt dynamically based on enabled tools, active skills, memory context, and user preferences
  • Context Compressor (agent/context_compressor.py) — Automatically summarises old messages when approaching the token limit
  • Prompt Caching (agent/prompt_caching.py) — Caches the system prompt for Anthropic models to reduce API costs by up to 80%
  • Credential Pool (agent/credential_pool.py) — Manages API keys across multiple providers, supporting rotation and fallback

6. Messaging Gateway — gateway/

A single GatewayRunner manages up to 29 messaging platforms simultaneously. Each platform has a dedicated adapter translating platform-specific message formats to/from the common internal format. The gateway maintains an LRU cache of up to 128 active AIAgent instances — one per user/channel session — and evicts idle sessions after 1 hour.

Data Flow: From Telegram Message to AI Response

1
Message Arrives via Telegram API

The Telegram adapter in gateway/platforms/telegram/ receives a webhook event and parses the message text, sender ID, and chat ID.

2
Session Lookup

The gateway looks up or creates an AIAgent instance for this sender, restoring their conversation history from SQLite.

3
Agent Loop

The message is passed to the AIAgent, which runs the full conversation loop (context assembly → LLM → tools → response).

4
Platform-Specific Formatting

The response is formatted for Telegram's Markdown variant (different from Slack's or Discord's) and any length limits are handled.

5
Message Sent + Session Saved

The formatted response is sent via Telegram API, and the session state (including new messages) is persisted to SQLite for the next turn.

Configuration System

The Tool Registry Pattern

All tools follow the same self-registration pattern. When run_agent.py starts, it imports all files in tools/ — each file calls registry.register() at import time. The result is a dynamic tool catalogue that can be extended simply by dropping a new Python file in the tools/ directory.

🔌
Extending Hermes with Custom Tools

Adding a new tool requires writing a single Python function decorated with @registry.register(). No changes needed to any other file. The tool automatically appears in the LLM's tool catalogue on the next restart.

Dangerous Command Detection

The tools/approval.py module contains a safeguard layer that detects potentially destructive terminal commands (e.g., rm -rf, DROP TABLE, format) and pauses execution to request human approval before proceeding. This is a critical safety mechanism for autonomous workflows.

The Session Search Engine

Every conversation Hermes has is stored in a SQLite database with FTS5 full-text search indexing. This means you can search across every conversation you've ever had:

# You can ask Hermes:
"What did I tell you about the Jenkins CI setup last month?"
# Hermes uses session_search to find the relevant conversation
# and recalls the exact context.

FTS5 supports proximity searches (words near each other), phrase matching, and ranking by relevance — not just simple keyword matching. The LLM then summarises the relevant excerpts in natural language.

The Self-Improving Skill Loop

The most distinctive feature of Hermes is its ability to autonomously create and improve skills from experience:

1
Complex Task Detected

When Hermes completes a multi-step task that took many tool calls, it identifies this as a "trajectory" worth saving.

2
Trajectory Saved

The sequence of steps — tools used, decisions made, output format — is saved via agent/trajectory.py.

3
Skill Synthesised

Hermes writes a Python script that encodes the workflow as a reusable skill with parameters (the variable parts you'd change each time).

4
Skill Installed + Available

The skill is saved to ~/.hermes/skills/ and immediately available as a slash command. Future invocations are fast and consistent.

5
Skill Improves Over Time

Each time the skill is used, Hermes can refine it based on the outcome — correcting errors, adding edge case handling, and updating outputs.

🌱
Why This Matters

Most AI assistants get no better over time — they're stateless. Hermes compounds: every task it completes either adds to its memory or becomes a reusable skill. After months of use, a Hermes instance personalised to your workflow is dramatically more efficient than a fresh one.

Session Persistence Across Platforms

Each user gets their own persistent session stored in SQLite. Importantly, sessions are platform-aware but conversation-continuous — your Hermes assistant remembers previous conversations regardless of whether you last spoke to it via Telegram or Slack.

The Cron Scheduler

Hermes can schedule tasks to run automatically using standard cron expressions. The scheduler (cron/scheduler.py) integrates with the gateway to deliver results to any configured platform:

# Examples of scheduled tasks
"0 9 * * 1-5"    → Run every weekday at 9am
"0 18 * * *"     → Run every day at 6pm
"*/30 * * * *"  → Run every 30 minutes

# Usage from chat:
/cron set "daily at 8am, check my emails and send me a summary"
/cron set "every Monday at 9am, pull our sales data and make a report"
⏰
Natural Language Scheduling

You don't need to know cron syntax. Tell Hermes "remind me every Friday afternoon to send the team update" and it converts it to the correct cron expression automatically.

ACP Integration — IDE Support

The acp_adapter/ module implements the Agent Client Protocol, allowing Hermes to be used directly inside VS Code, Zed, and JetBrains IDEs. The adapter runs as a background server that editors communicate with via JSON-RPC — you get Hermes' full tool ecosystem (web search, file management, code execution) inside your editor without switching to the terminal.

Building a Custom Tool

Creating a new tool for Hermes requires a single Python file in the tools/ directory:

# tools/my_custom_tool.py
from tools.registry import registry
import httpx

@registry.register(
    name="get_stock_price",
    description="Get the current stock price for a ticker symbol",
    parameters={
        "ticker": {
            "type": "string",
            "description": "Stock ticker symbol, e.g. AAPL"
        }
    }
)
async def get_stock_price(ticker: str) -> str:
    async with httpx.AsyncClient() as client:
        resp = await client.get(
            f"https://api.example.com/price/{ticker}"
        )
    data = resp.json()
    return f"${data['price']} USD ({ticker})"

That's it. On next run, the tool appears in the LLM's tool catalogue automatically.

Building a Custom Skill

Skills live in ~/.hermes/skills/<skill-name>/ and consist of:

# ~/.hermes/skills/weekly-report/SKILL.md
---
name: weekly-report
description: Generate a weekly business performance report
trigger: /weekly-report
---
Pull the last 7 days of sales data, compare to the previous week,
highlight top performers, and format as a Slack-ready summary.

# ~/.hermes/skills/weekly-report/main.py
import sys

args = sys.argv[1:]
week = args[0] if args else "current"

print(f"Generating weekly report for: {week}")
print("Focus on: revenue, top SKUs, and week-over-week delta.")
